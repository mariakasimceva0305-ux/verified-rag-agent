# Verified RAG Agent

Локальный RAG-агент с переформулировкой запроса, поиском, переранжированием, генерацией ответа, цитированием и проверкой соответствия ответа найденным источникам.

## Задача
Нужно уменьшить число ответов, которые плохо опираются на найденные документы, и сделать выход системы проверяемым за счёт цитирования и отдельного этапа верификации.

## Архитектура решения
```text
Вопрос
  │
  ▼
Переформулировка запроса
  │
  ▼
Поиск документов
  │
  ▼
Переранжирование
  │
  ▼
Генерация ответа
  │
  ▼
Проверка ответа
  │
  ▼
Ответ с цитированием
```

## Структура репозитория
- `configs/` — параметры запуска;
- `src/` — основной код;
- `eval/` — сценарии и материалы оценки;
- `docs/` — дополнительные пояснения;
- `tests/` — тесты;
- `examples/` — примеры запуска.

## Метрики
- `HitRate@K`
- `MRR`
- задержка ответа (`p50`, `p95`)
- доля ответов, подтверждённых найденными источниками

## Результаты
В итоговом README нужно показать:
- изменение `HitRate@K` и `MRR` после переформулировки запроса и переранжирования;
- изменение доли подтверждённых ответов после включения этапа проверки;
- задержку базового и полного контура.

Формат записи:
- `Переформулировка запроса и переранжирование повысили MRR с ... до ...`
- `Этап проверки увеличил долю подтверждённых ответов с ... до ...`

## Запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m src
```

## Ограничения и развитие
- Итоговое качество определяется качеством поиска и полнотой корпуса;
- дополнительные проверки увеличивают задержку;
- следующий шаг — вынести сравнение режимов в отдельный отчёт с воспроизводимыми числами.
