# Verified RAG Agent

Локальный RAG-контур с переформулировкой запроса, поиском, переранжированием, генерацией ответа, цитированием и проверкой соответствия ответа найденным источникам.

## Назначение

Снизить число ответов, которые слабо опираются на найденные документы, и сделать выдачу проверяемой за счёт ссылок на источники и отдельного этапа проверки.

## Архитектура

```text
Вопрос
  │
  ▼
Переформулировка запроса
  │
  ▼
Поиск документов
  │
  ▼
Переранжирование
  │
  ▼
Генерация ответа
  │
  ▼
Проверка ответа по источникам
  │
  ▼
Ответ с цитированием
```

## Что реализовано

- переформулировка запроса перед поиском;
- поиск и переранжирование найденных документов;
- формирование ответа только по найденному контексту;
- цитирование использованных источников;
- отдельный этап проверки ответа;
- материалы и сценарии оценки.

## Структура репозитория

- `configs/` — конфигурации запуска;
- `data/` — данные и демонстрационные материалы;
- `docs/` — пояснения по архитектуре;
- `eval/` — сценарии и артефакты оценки;
- `examples/` — примеры использования;
- `scripts/` — сценарии запуска;
- `src/` — основной код;
- `tests/` — тесты.

## Оценка качества

Основные метрики:

- `HitRate@K`
- `MRR`
- доля ответов, подтверждённых найденными источниками (при наличии отдельного расчёта в сценарии оценки)
- задержка ответа: `p50`, `p95`

Сравнения по контурам:

- поиск без переформулировки и переранжирования (`configs/baseline.yaml`);
- поиск с переформулировкой и переранжированием и этапом проверки (`configs/improved.yaml`).

## Результаты

Оценка: `python scripts/run_eval.py` после `ingest` и `build_index` для выбранной конфигурации, набор `eval/questions.jsonl` (16 вопросов). Запуск из корня репозитория с `PYTHONPATH=.` (или эквивалентом).

| Конфигурация | HitRate@K | MRR | latency_p50_ms | latency_p95_ms |
|--------------|-----------|-----|----------------|----------------|
| `configs/baseline.yaml` | 0.0 | 0.0 | 7.16 | 9.29 |
| `configs/improved.yaml` | 0.0 | 0.0 | 4.93 | 6.47 |

Нулевой `HitRate@K` на этом наборе отражает фактический прогон: при изменении корпуса, индекса или эталонных `gold_doc_ids` значения пересчитываются.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# пример: подготовка и оценка для одной конфигурации
export PYTHONPATH=.
python scripts/ingest.py --config configs/baseline.yaml
python scripts/build_index.py --config configs/baseline.yaml
python scripts/run_eval.py --config configs/baseline.yaml

python scripts/demo.py --config configs/improved.yaml
```

В среде Windows перед запуском скриптов задают переменную окружения `PYTHONPATH` на корень репозитория (аналог `export PYTHONPATH=.`).

## Ограничения

- качество ответа определяется полнотой корпуса и качеством поиска;
- дополнительные этапы повышают задержку;
- для стабильного сравнения режимов сохраняют отдельные отчёты по каждому сценарию оценки.
