# Verified RAG Agent

## Кратко
Система ответов по документам, в которой после обычного RAG-контура добавлен отдельный этап проверки ответа по найденным источникам. Такой подход снижает риск неподтверждённых ответов и делает поведение системы более устойчивым на сложных запросах.

## Задача
Обычный RAG-контур помогает опереться на найденные документы, но сам по себе не гарантирует, что итоговый ответ действительно подтверждён источниками. В этом проекте задача решается в несколько этапов: переписывание запроса, поиск, переранжирование, формирование чернового ответа и отдельная проверка ответа по найденному контексту.

## Что улучшено
- добавлен этап переписывания запроса для повышения устойчивости поиска;
- добавлено переранжирование найденных фрагментов перед генерацией ответа;
- добавлен этап проверки итогового ответа по источникам;
- снижено число ответов, которые формально звучат убедительно, но не подтверждаются найденным контекстом.

## Архитектура
```text
[запрос пользователя]
          |
          v
[переписывание запроса]
          |
          v
[поиск фрагментов]
          |
          v
[переранжирование]
          |
          v
[черновой ответ]
          |
          v
[проверка по найденным источникам]
          |
          v
[итоговый ответ с опорой на контекст]
```

## Структура репозитория
```text

src/       основная логика системы
scripts/   сценарии запуска и подготовки
eval/      оценка качества

```

## Критерии качества
Система оценивается по трём группам критериев:

1. **Качество поиска**
   - HitRate@K
   - MRR

2. **Качество ответа**
   - доля ответов, подтверждённых найденным контекстом;
   - устойчивость на запросах с шумом, пропусками и неоднозначными формулировками.

3. **Системные характеристики**
   - задержка ответа;
   - воспроизводимость результата при одинаковой конфигурации.

## Ключевой вывод
Добавление этапа проверки после поиска, переранжирования и генерации повышает надёжность итогового ответа: система реже возвращает неподтверждённые формулировки и лучше отделяет достаточный контекст от недостаточного.

## Запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m src
```

## Ограничения
- качество зависит от полноты корпуса и качества поиска;
- этап проверки не заменяет поиск, а работает поверх него;
- на шумных и слишком коротких корпусах выигрыш от проверки ограничен.
